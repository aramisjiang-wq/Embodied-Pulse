## 一、用户习惯分析

### 1.1 用户使用 HuggingFace 的目的

用户使用 HuggingFace 主要为了获取三类资产：

- **论文（Papers）**
- **模型（Models）**
- **数据集（Datasets）**

部分作者（如 [GD-ML](https://huggingface.co/GD-ML)）会定期发布 papers、models 或 datasets。

### 1.2 典型使用模式

| 模式 | 描述 | 典型用户 |
|------|------|----------|
| **作者驱动** | 关注机构/作者（如 GD-ML、meta-llama、UC-Berkeley-AI），定期查看其新论文、模型、数据集 | 研究者、工程师 |
| **任务驱动** | 按任务类型（robotics、object-detection 等）或关键词搜索，找到即用 | 工程落地、复现实验 |
| **资产驱动** | 围绕论文→代码→模型→数据集这条链路，跨平台串联使用 | 做复现、做调研的人 |

### 1.3 核心结论

1. **作者订阅很重要**：很多机构（如 GD-ML）会同时发布 papers、models、datasets，用户更习惯「跟人」而不是「跟单个模型」。
2. **跨平台串联**：用户会在 arXiv 看论文 → GitHub 找代码 → HuggingFace 找模型/数据集，需要在一个地方能串联这些资产。
3. **收藏与跟进**：用户会收藏「可能用得上」的模型/数据集，并希望在有更新时收到提醒。

## 二、当前页面能力评估

### 2.1 已有能力

- 订阅作者（`subscribeAuthor`）
- 提交模型（粘贴 HF URL 解析后提交）
- 收藏单个模型
- 关注单个模型（获取更新通知）
- 内容类型筛选：模型 / 数据集 / 空间
- 任务类型筛选、排序、搜索

### 2.2 不足与缺口

| 缺口 | 说明 |
|------|------|
| 订阅作者后的聚合视图 | 订阅了作者，但没有「我订阅作者的新动态」聚合入口 |
| 提交入口仅限模型 | 用户同样需要提交 datasets，当前入口文案和流程偏模型 |
| 收藏入口分散 | 收藏能力已有，但缺少「我的收藏」聚合视图 |
| 论文与模型/数据集关联 | 暂未打通 papers 与 models/datasets 的关联展示 |

## 三、针对性优化建议

### 3.1 强化「订阅作者」体验

- 在页面顶部或侧边增加「我订阅作者的新动态」入口。
- 展示：作者名 + 最近更新的 models/datasets/papers（如有）列表。
- 支持按作者筛选当前列表，例如「只看 GD-ML 的内容」。

### 3.2 统一「提交」入口，支持模型和数据集

- 将「提交模型」改为「提交项目」或「添加模型/数据集」。
- 支持提交 model 和 dataset 两种 URL，例如：
  - `https://huggingface.co/meta-llama/Llama-3.2-1B`
  - `https://huggingface.co/datasets/xxx`
- 后端已有 `contentType: model | dataset | space`，前端只需在提交时区分并传对类型。

### 3.3 收藏与「我的资产」视图

- 在导航或个人中心增加「我的收藏」或「我的 HuggingFace 资产」。
- 支持按类型筛选：模型 / 数据集 / 空间。
- 展示收藏时间、最近更新时间，方便用户判断是否要重新查看。

### 3.4 作者维度的聚合视图

- 在作者详情或作者列表中，展示该作者下的 models、datasets、papers 数量。
- 若系统有 papers 数据，可展示「论文 → 关联模型/数据集」的链接。

### 3.5 信息架构上的小调整

- 左侧「内容类型」中，把「模型」「数据集」放在更显眼的位置。
- 统计文案从「个 AI 模型」改为「个模型/数据集」，与实际数据一致。

## 四、实施优先级建议

| 优先级 | 改动 | 理由 |
|--------|------|------|
| P0 | 提交入口支持 dataset | 用户有提交数据集的需求，改动小 |
| P0 | 订阅作者的新动态视图 | 直接提升订阅价值 |
| P1 | 收藏入口与「我的收藏」视图 | 已有收藏能力，缺聚合入口 |
| P1 | 按作者筛选列表 | 方便快速看某作者的全部内容 |
| P2 | 论文与模型/数据集的关联 | 依赖 papers 数据与关联关系，实现成本较高 |

---

## 五、小结

- **用户习惯**：以作者为中心（关注机构/作者）和以任务为中心（按任务/关键词搜索）并存，且会跨平台串联论文、代码、模型、数据集。
- **当前页面**：订阅作者、收藏、关注、提交模型等能力已有，但缺少「订阅作者的新内容聚合」和「提交数据集」。
- **建议优先**：
  1. 提交入口支持 dataset
  2. 增加「我订阅作者的新动态」视图
  3. 增加「我的收藏」聚合入口

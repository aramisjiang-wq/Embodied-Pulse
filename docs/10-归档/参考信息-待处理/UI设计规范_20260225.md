## 一、规范目标与范围

### 1.1 规范目标

- 统一用户端与管理端视觉语言，降低实现偏差
- 在“极简、高信息密度、快速浏览”的前提下，允许少量高质量视觉特效
- 保障可访问性（WCAG 2.1 AA）与性能预算，不因特效牺牲可用性

### 1.2 页面范围（与 PRD/SPEC 对齐）

用户端共 16 个页面：

- 公开页面 10 个：`/`、`/papers`、`/papers/[id]`、`/videos`、`/videos/[id]`、`/resources`、`/resources/[id]`、`/news`、`/news/[id]`、`/jobs`
- 认证页面 3 个：`/login`、`/register`、`/forgot-password`
- 登录后页面 3 个：`/profile`、`/favorites`、`/subscriptions`

管理端共 12 个页面，沿用 `管理端原型图_20260225.md`。

## 二、视觉风格与设计原则

### 2.1 风格定位

- 关键词：`专业`、`克制`、`高效`、`科技感`
- 视觉策略：以内容卡片为主，不依赖大面积装饰
- 允许特效：仅用于首页 Hero、卡片悬浮反馈、状态过渡，不用于正文阅读区

### 2.2 设计原则

- 一致性：相同语义使用相同组件、颜色、交互反馈
- 可读性：正文优先，文本对比度达标
- 可扫描性：列表页强调分组、标签、关键指标
- 可恢复性：所有危险操作必须可撤销或二次确认

## 四、组件规范（基础 + 业务）

### 4.1 技术栈分工（避免冲突）

- `Ant Design 5`：表单、表格、弹窗、日期选择、通知等中后台强交互组件
- `Tailwind CSS`：页面布局、间距、响应式、轻量视觉样式
- 约束：不直接覆盖 AntD 内联样式；统一通过 token/theme 覆盖

### 4.2 基础组件

- Button：`primary / default / text / link`；高度 `32 / 40 / 48`
- Input：默认高 `40`，错误态必须展示错误文案（不仅红边）
- Card：默认白底 + 1px 边框，悬浮态上浮 `-2px`
- Tag：按内容类型色映射，不允许自行扩展新主色

### 4.3 业务组件

- `FeedCard`：统一信息流卡片骨架，支持五类内容
- `TypeBadge`：内容类型标签组件，使用固定映射
- `SourcePill`：资源多来源胶囊（GitHub/HuggingFace 等）
- `SubscriptionRuleItem`：订阅规则项（关键词、类型、频率、开关）

## 五、动效与特效白名单

### 5.1 动效时长与缓动

| 场景 | 时长 | 缓动 |
|------|------|------|
| Hover反馈 | 120-160ms | `ease-out` |
| 常规过渡 | 180-240ms | `ease-in-out` |
| 进入动画 | 260-320ms | `cubic-bezier(0.22,1,0.36,1)` |

### 5.2 允许的特效组件（用户同意可使用）

1. `GradientMeshHero`：仅首页顶部，静态渐变 + 缓慢位移动画
2. `GlassStatCard`：仅首页关键统计卡（最多 3 个）
3. `HoverGlowCard`：卡片轻微辉光，透明度 <= 0.18
4. `NumberTicker`：统计数字滚动，时长 <= 600ms

### 5.3 禁止项

- 禁止全页粒子特效、视频背景、持续闪烁动画
- 禁止滚动视差强位移（影响阅读）
- 禁止高频阴影/模糊动画导致 GPU 持续高占用

### 5.4 降级策略

```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 1ms !important;
    transition-duration: 1ms !important;
  }
}
```

## 六、页面级布局规范

### 6.1 用户端

- Header：`64px`
- 主内容容器：`max-width: 1200px; margin: 0 auto; padding: 24px 16px`
- Feed 列表：移动端 1 列，桌面端 1 列（强调阅读）
- 分类页卡片：`1 / 2 / 3` 列响应式切换（根据内容密度）

### 6.2 管理端

- Header：`64px`
- Sidebar：`220px`（可折叠为 `80px`）
- 主区：表格优先，顶部固定筛选区

### 6.3 空状态与错误状态

- 空状态：图标 + 1 行说明 + 1 个主操作按钮
- 错误状态：说明 + 重试按钮 + 错误ID（便于排查）

## 七、响应式与可访问性

### 7.1 断点

- `xs`: `<576px`
- `sm`: `>=576px`
- `md`: `>=768px`
- `lg`: `>=992px`
- `xl`: `>=1200px`
- `xxl`: `>=1600px`

### 7.2 可访问性最低要求

- 文本对比度：普通文本 >= `4.5:1`
- 键盘可达：关键交互支持 `Tab` 访问
- Focus 样式：必须可见，不可去掉 outline 后不补
- 表单：错误提示与输入控件 `aria-describedby` 关联

## 八、工程落地规范

### 8.1 命名与目录建议

- `components/ui/*`：基础组件
- `components/biz/*`：业务组件
- `styles/tokens.css`：设计令牌
- `styles/motion.css`：动效 token

### 8.2 性能预算

- 首屏 LCP（4G）：<= 2.5s
- CLS：<= 0.1
- 首屏 JS（gzip）：<= 220KB
- 单页面自定义动画组件：<= 3 个

### 8.3 质量门禁

- 视觉回归：关键页面截图对比
- a11y 扫描：无严重级别问题
- 动效审查：满足“白名单 + 降级策略”

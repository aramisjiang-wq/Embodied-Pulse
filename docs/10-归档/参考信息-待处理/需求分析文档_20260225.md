### 1.2 核心问题识别

| 问题类型 | 具体表现 | 根本原因 | 影响 |
|----------|----------|----------|------|
| **功能臃肿** | 57个页面、100+接口、22个模型 | 缺乏聚焦，什么都想做 | 维护成本高、用户认知负担重 |
| **定位模糊** | 聚合+社区+订阅+招聘 | 没有明确核心价值 | 核心价值不突出 |
| **用户路径复杂** | 多入口、多订阅类型、多管理页面 | 缺乏用户视角设计 | 用户迷失、转化率低 |
| **技术债务** | 代码重复、类型不完整、测试不足 | 快速迭代无规范 | 迭代变慢、Bug增多 |
| **性能问题** | 首屏加载~2s、LCP~3s | 功能过多、优化不足 | 用户体验差 |

### 1.3 马斯克视角批判

**问题：这个产品为什么这么复杂？**

```
┌─────────────────────────────────────────────────────────────────────────────┐

│                                                                             │

## 二、第一性原理分析

### 2.1 产品本质

**马斯克提问：这个产品的最基本构成是什么？**

```
产品本质 = 信息聚合 + 领域过滤 + 高效获取

最简形式：
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                             │
│   输入：用户兴趣（具身智能领域）                                              │
│                                                                             │
│   处理：                                                                     │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │  1. 聚合：从多个数据源收集信息                                        │  │
│   │  2. 过滤：按领域关键词筛选                                            │  │
│   │  3. 排序：按相关性/热度/时间排序                                      │  │
│   │  4. 展示：以统一格式呈现                                              │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
│   输出：高价值信息流                                                         │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.2 用户核心需求

**马斯克提问：用户真正需要什么？**

| 需求层级 | 需求描述 | 痛点强度 | 当前满足程度 | Pro版目标 |
|----------|----------|----------|--------------|-----------|
| **生存需求** | 获取领域最新进展，保持竞争力 | 🔴 极高 | ⚠️ 困难 | ✅ 10分钟/天 |
| **效率需求** | 快速找到需要的资源（论文/代码/模型） | 🔴 极高 | ⚠️ 低效 | ✅ 1分钟找到 |
| **职业需求** | 了解行业动态、发现工作机会 | 🔴 极高 | ⚠️ 分散 | ✅ 一站获取 |
| **连接需求** | 与同行交流、发现合作机会 | 🟡 中等 | ⚠️ 分散 | 🔄 MVP后迭代 |
| **成长需求** | 系统性学习、职业发展 | 🟡 中等 | ⚠️ 无序 | 🔄 MVP后迭代 |

### 2.3 "10倍改进"检验

**马斯克提问：这个产品是否比现有方案好10倍？**

| 维度 | 现有方案 | 老项目 | Pro版目标 | 改进程度 |
|------|----------|--------|-----------|----------|
| 时间效率 | 1小时/天 | 30分钟/天 | **10分钟/天** | **6x** |
| 信息精准度 | <50% | 70% | **>95%** | **2x** |
| 信息遗漏率 | 30% | 15% | **<3%** | **10x** |
| 领域聚焦度 | <5% | 80% | **100%** | **20x** |
| 用户认知负担 | 高（多平台） | 中（功能多） | **低（极简）** | **显著改善** |

### 2.4 核心价值主张

**一句话价值主张**：

> 具身智能领域的一站式信息聚合平台，让从业者用10分钟完成原本1小时的信息获取工作。

**核心价值量化**：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           用户价值计算                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   时间价值                                                                   │
│   ├── 节省时间：50分钟/天                                                    │
│   ├── 时间价值：100元/小时（研究者平均时薪）                                   │
│   ├── 每月工作日：22天                                                       │
│   └── 月度价值 = 50分钟 × 100元/60 × 22天 = 1,833元/月                       │
│                                                                             │
│   信息价值（难以量化）                                                        │
│   ├── 不再错过重要论文：无价                                                  │
│   ├── 更快找到代码实现：加速研发进度                                          │
│   ├── 更好的技术选型：降低试错成本                                            │
│   ├── 行业动态及时获取：决策参考                                              │
│   └── 工作机会发现：职业发展                                                  │
│                                                                             │
│   结论：保守估计年度价值 > 20,000元                                           │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 三、核心需求定义

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           需求优先级矩阵                                      │

│   │  ✅ 资源聚合              - 按类型分类（模型/数据集/代码等）          │  │
│   │  ✅ 新闻聚合              - 行业动态（人工筛选）                      │  │
│   │  ✅ 招聘聚合              - 工作机会（列表+外链）                     │  │
│   │  ✅ 搜索功能              - 核心入口                                  │  │
│   │  ✅ 收藏功能              - 用户粘性                                  │  │
│   │  ✅ 关键词订阅            - 不遗漏信息                                │  │
│   └─────────────────────────────────────────────────────────────────────┘  │

│   高价值、低频使用（重要需求 - MVP简化版）                                    │

### 3.2 核心内容模块

| 模块 | 用户价值 | 使用频率 | MVP实现 |
|------|----------|----------|---------|
| **论文聚合** | 追踪最新研究、找代码复现 | 每日 | 列表页 + 详情页 + 搜索 + 订阅 |
| **视频聚合** | 学习入门、技术讲解 | 每日 | 列表页 + 详情页 + 搜索 + 订阅 |
| **资源聚合** | 找模型/数据集/代码实现 | 每周 | 统一列表页 + 类型筛选 + 多来源 |
| **新闻聚合** | 了解行业动态、决策参考 | 每日 | 列表页 + 人工筛选 |
| **招聘聚合** | 发现工作机会、职业发展 | 每周 | 列表页 + 外链跳转 |

### 3.3 用户故事

**用户故事1：追踪最新论文**

```
作为一名 具身智能领域的研究者
我想要 每天花10分钟了解领域最新论文
以便于 保持研究方向的领先性

验收标准：
- 首页展示最新论文，按时间排序
- 可以按关键词筛选论文
- 可以订阅关键词，自动推送新论文
- 可以一键查看论文详情和代码链接
```

**用户故事2：找资源实现**

```
作为一名 机器人工程师
我想要 快速找到模型、数据集或代码实现
以便于 参考实现或直接使用

验收标准：
- 资源页面按类型分类（模型/数据集/代码等）
- 可以按类型筛选资源
- 资源详情页显示多个来源链接（GitHub、HuggingFace等）
```

**用户故事3：了解行业动态**

```
作为一名 从业者
我想要 每天了解具身智能领域的行业动态
以便于 做出更好的职业和投资决策

验收标准：
- 新闻列表页展示精选行业动态
- 新闻由管理员人工筛选，保证质量
- 可以按类型筛选新闻（融资、产品、政策等）
```

**用户故事4：发现工作机会**

```
作为一名 求职者
我想要 发现具身智能领域的工作机会
以便于 找到合适的工作

验收标准：
- 招聘列表页展示相关职位
- 点击职位直接跳转到招聘平台
- 可以按地点、薪资筛选职位
```

### 4.3 MVP核心原则

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           MVP核心原则                                         │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   原则1：用户视角分类                                                        │
│   ├── 资源按类型分类，而非来源                                                │
│   ├── 用户关心"找模型"，不关心"模型在哪个平台"                                │
│   └── 来源只是数据字段，方便扩展                                              │
│                                                                             │
│   原则2：极简主义                                                            │
│   ├── 每个功能必须有明确的用户价值                                            │
│   ├── 砍掉所有"有了更好"的功能                                                │
│   └── 只保留"没有不行"的功能                                                  │
│                                                                             │
│   原则3：聚焦核心                                                             │
│   ├── 论文、视频、资源、新闻、招聘是核心五件套                                │
│   ├── 核心功能要做到90分                                                      │
│   └── 其他功能都是辅助                                                        │
│                                                                             │
│   原则4：快速验证                                                             │
│   ├── MVP目标是验证核心价值                                                   │
│   ├── 不追求完美，追求快速上线                                                 │
│   └── 基于用户反馈迭代                                                        │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

│    id: string                                                               │
